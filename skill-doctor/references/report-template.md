## 报告规则

- **问题区顺序**：按维度/因果链同序——🧭理解 → 🔗执行 → 🧱经济（与评分卡一致）。
- **先引用，后评判**：每个维度必须给原文/原文件引用。
- 理解、执行、经济三维必须展示子项得分。
- 执行稳定性的每条发现带证据标签：`[实跑]`/`[静态]`/`[核对]`。
- 前 3 优先修复按"提分潜力 × 重要性"排序，给具体改法和预计提分。
- 5/5 的维度写"无需修改。"
- 逐文件审计小结覆盖整包每个文件，含 SKILL.md 行数。

## 完成前验证

输出报告前逐项检查：
- [ ] 4 维均已评分，每个分有原文/原文件引用支撑
- [ ] 理解、执行、经济的子项得分已展示
- [ ] 执行稳定性发现都带证据标签，且注明实跑/静态规模
- [ ] 整包每个文件都进了逐文件审计小结
- [ ] 总分计算正确（触发×2 + 理解×6 + 执行×7 + 经济×5）
- [ ] 已输出前 3 优先修复 + 快速修复
